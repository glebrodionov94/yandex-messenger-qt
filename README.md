# Yandex Messenger Qt

[![Build and release](https://github.com/glebrodionov94/yandex-messenger-qt/actions/workflows/build-release.yml/badge.svg)](https://github.com/glebrodionov94/yandex-messenger-qt/actions/workflows/build-release.yml)

Неофициальная Qt 6-оболочка для веб-версии Яндекс Мессенджера на Linux. Это не отдельный клиент, а надстройка над сайтом, поэтому её поведение зависит от текущей версии веб-интерфейса.

## Возможности

- отдельное окно с веб-версией Яндекс Мессенджера;
- сохранение авторизации через профиль Qt WebEngine;
- одиночный экземпляр приложения для одного пользователя Linux;
- запуск в системный трей с восстановлением окна по клику;
- монохромная иконка трея и красный индикатор при новом HTML5-уведомлении;
- HTML5-уведомления используются как индикатор новых сообщений, но это не точный счётчик непрочитанных;
- автозапуск через `~/.config/autostart/yandex-messenger-qt.desktop`;
- запуск скрытым в трее;
- безопасный режим, который отключает визуальные исправления страницы;
- скрытие левой панели сервисов Яндекса;
- удаление внешних отступов страницы;
- очистка cookies и HTTP-кэша приложения;
- сброс профиля приложения без затрагивания обычного Яндекс Браузера;
- копирование диагностической информации без личных данных;
- диалог «О программе» со ссылками на репозиторий и страницу ошибок.

## Установка готового .deb

Для Ubuntu и Kubuntu рекомендуется устанавливать `.deb` из раздела Releases.

```bash
sudo apt install ./<package>.deb
```

## Совместимость пакетов

| Файл | Поддерживаемая система |
|---|---|
| `*_ubuntu-24.04_amd64.deb` | Ubuntu/Kubuntu 24.04 LTS |
| `*_ubuntu-26.04_amd64.deb` | Ubuntu/Kubuntu 26.04 LTS |

Пояснение: `.deb` пакеты используют системные библиотеки Qt, поэтому необходимо скачивать пакет, собранный для вашей версии Ubuntu (24.04 или 26.04). Один пакет не гарантированно будет работать на обеих версиях из-за разницы в названиях библиотек (например, `libqt6gui6t64` vs `libqt6gui6`).

Для Wayland и демонстрации экрана понадобятся PipeWire и портал KDE:

```bash
sudo apt install xdg-desktop-portal xdg-desktop-portal-kde pipewire
```

## Сборка из исходников

Для Ubuntu или Kubuntu:

```bash
sudo apt update
sudo apt install build-essential cmake ninja-build qt6-base-dev qt6-webengine-dev dpkg-dev

git clone https://github.com/glebrodionov94/yandex-messenger-qt.git
cd yandex-messenger-qt
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
cd build
cpack -G DEB
```

После сборки в каталоге `build/` появится бинарник приложения и `.deb` пакет.

## Запуск

Запустите бинарник из каталога сборки:

```bash
./build/yandex-messenger-qt
```

Если приложение уже запущено, второй старт не создаёт новое окно и просто активирует первый экземпляр.

## Wayland и демонстрация экрана

Под Wayland приложение работает через Qt WebEngine и стандартные порталы xdg-desktop-portal. Для демонстрации экрана нужны PipeWire и `xdg-desktop-portal-kde`.

Если используется KDE Plasma Wayland, убедитесь, что портал установлен и активен:

```bash
sudo apt install xdg-desktop-portal-kde pipewire
```

## Автозапуск

В меню трея есть переключатель `Запускать при входе в систему`. Он создаёт или удаляет только файл:

```bash
~/.config/autostart/yandex-messenger-qt.desktop
```

При включении записывается реальный путь к исполняемому файлу. При выключении удаляется только файл, созданный приложением.

## Системный трей

Приложение рассчитано на работу из системного трея. Закрытие окна прячет приложение в трей, а не завершает его.

В меню трея доступны:

- открытие окна;
- перезагрузка страницы;
- запуск скрытым в трее;
- автозапуск;
- безопасный режим визуальных исправлений;
- отдельные переключатели скрытия боковой панели и удаления отступов;
- очистка cookies;
- сброс профиля;
- копирование диагностики;
- окно «О программе»;
- выход.

## Безопасный режим

Переключатель `Включить визуальные исправления страницы` отключает все JavaScript-исправления интерфейса и перезагружает страницу.

Если режим выключен, приложение не внедряет скрипты для скрытия левой панели и удаления внешних отступов. При этом исходный вид страницы восстанавливается.

Отдельные переключатели:

- `Скрыть левую панель сервисов`;
- `Убирать внешние отступы страницы`.

Они активны только при включённых визуальных исправлениях.

## Очистка cookies

Пункт `Очистить cookies` запрашивает подтверждение, очищает cookies и HTTP-кэш Qt WebEngine и перезагружает страницу.

После очистки может потребоваться повторный вход.

## Сброс профиля

Пункт `Сбросить профиль приложения` запрашивает подтверждение, сообщает о завершении приложения и затем удаляет только каталоги профиля и кэша текущего приложения.

Это не затрагивает обычный Яндекс Браузер и не удаляет произвольные каталоги. После сброса при следующем запуске создаётся чистый профиль.

## Диагностика

Пункт `Скопировать диагностическую информацию` помещает в буфер обмена только технические сведения:

- версию приложения;
- версию Qt;
- тип сессии;
- тип рабочего стола;
- выбранный платформенный плагин;
- путь профиля и кэша;
- состояние визуальных исправлений;
- состояние скрытия панели;
- состояние компактного режима;
- автозапуск;
- запуск скрытым.

Cookies, токены, история чатов и другие личные данные не копируются.

## Ограничения

- Это неофициальный проект и не официальный клиент Яндекса.
- HTML5-уведомления используются как источник индикатора новых сообщений, но это не точный счётчик непрочитанных.
- Визуальные исправления страницы зависят от структуры сайта и могут потребовать обновления после изменений веб-интерфейса.
- Архив с бинарником не является полностью статической сборкой.
- Для Ubuntu и Kubuntu предпочтительнее `.deb`, потому что APT сам разрешает зависимости.
- Для разрешений камеры, микрофона и демонстрации экрана остаётся явное подтверждение пользователя.

## Публикация релиза

Сборка настроена в GitHub Actions в файле `.github/workflows/build-release.yml`.

Workflow запускается при push в `main`, при pull request и вручную. GitHub Release создаётся только при теге вида `v*`.

Локально релиз можно подготовить так:

```bash
git push origin main
git tag -a v0.2.0 -m "Release v0.2.0"
git push origin v0.2.0
```
